# FanPick - 抖音小程序产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
**FanPick（粉丝选品）**

### 1.2 产品定位
一款帮助抖音视频创作者了解粉丝购物偏好的工具型小程序。粉丝可以浏览创作者橱窗中的产品并标记"想要"，创作者可以查看投票排行，精准上架粉丝最期待的商品。

### 1.3 目标用户
| 角色 | 描述 |
|------|------|
| 创作者（Creator） | 抖音视频制作者，拥有商品橱窗，需要了解粉丝偏好 |
| 粉丝（Fan） | 创作者的粉丝，希望创作者推荐自己喜欢的产品 |

### 1.4 核心价值
- **创作者**：减少选品试错成本，提升橱窗转化率
- **粉丝**：参与感更强，能买到真正想买的产品

## 2. 功能需求

### 2.1 粉丝端

#### P0 - 必须实现
| 功能 | 描述 |
|------|------|
| 浏览产品列表 | 查看创作者橱窗中的所有产品，展示图片、名称、价格 |
| 点击"想要" | 对喜欢的产品进行投票，每人每产品一次 |
| 取消"想要" | 可以撤回投票 |
| 查看排行榜 | 查看所有产品按"想要"数排序的榜单 |

#### P1 - 重要
| 功能 | 描述 |
|------|------|
| 分享小程序 | 将小程序分享给其他粉丝参与投票 |
| 产品详情 | 查看产品详细信息、跳转抖音商城 |
| 投票结果查看 | 实时查看当前投票排名 |

#### P2 - 可选
| 功能 | 描述 |
|------|------|
| 评论/留言 | 粉丝可以在产品下留言表达需求 |
| 站内通知 | 投票结果变化时通知粉丝 |

### 2.2 创作者端

#### P0 - 必须实现
| 功能 | 描述 |
|------|------|
| 绑定橱窗 | 授权关联自己的抖音商品橱窗 |
| 同步商品 | 一键同步橱窗商品到小程序 |
| 查看投票数据 | 查看各产品的"想要"数量排行 |
| 粉丝画像概览 | 查看参与投票的粉丝数量统计 |

#### P1 - 重要
| 功能 | 描述 |
|------|------|
| 分享投票页 | 生成投票卡片/链接，发布到视频或评论区 |
| 数据导出 | 导出投票数据为表格 |
| 批量管理 | 批量上下架投票产品 |

## 4. 数据模型

### 4.1 用户表 (users)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| openid | VARCHAR | 抖音用户唯一标识 |
| nickname | VARCHAR | 昵称 |
| avatar | VARCHAR | 头像URL |
| role | ENUM | fan / creator |
| creator_id | VARCHAR | 抖音创作者ID（创作者专用） |
| created_at | DATETIME | 注册时间 |

### 4.2 产品表 (products)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| creator_id | INT | 关联创作者 |
| product_id | VARCHAR | 抖音商品ID |
| title | VARCHAR | 商品名称 |
| image | VARCHAR | 商品图片 |
| price | DECIMAL | 价格（分） |
| detail_url | VARCHAR | 商品详情链接 |
| status | ENUM | active / inactive |
| want_count | INT | "想要"计数（冗余字段） |
| created_at | DATETIME | 创建时间 |

### 4.3 投票表 (wants)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| user_id | INT | 投票用户 |
| product_id | INT | 投票产品 |
| created_at | DATETIME | 投票时间 |
| UNIQUE | | (user_id, product_id) |

## 5. 技术方案

### 5.1 前端
- **框架**：抖音小程序（TTML + TTSS + JavaScript）
- **UI 组件**：自定义组件 + 抖音内置组件
- **网络请求**：tt.request API

### 5.2 后端
- **运行时**：Node.js + Express
- **数据库**：MySQL 8.0
- **认证**：抖音小程序登录 code2session
- **API 风格**：RESTful

### 5.3 部署
- 后端部署于云服务器（阿里云/腾讯云）
- 数据库使用云数据库 RDS
- 小程序通过抖音开发者工具提交审核

## 7. 风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|---------|
| 抖音橱窗 API 权限受限 | 无法自动同步商品 | 支持手动添加商品作为备选方案 |
| 小程序审核不通过 | 延迟上线 | 提前准备资质文件，预留审核时间 |
| 刷票/作弊 | 数据失真 | 限制每人每商品一次，需要登录验证 |
| 高并发投票 | 服务器压力 | Redis 缓存投票数据，异步落库 |
